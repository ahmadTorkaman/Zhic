## 3. Schema additions needed

### Designs collection

Add one field. Collection name unchanged (collection is generic; URL is `/bedroom-set/*`).

```ts
occupancies: {
  type: 'select',
  hasMany: true,
  options: ['baby', 'teen', 'double', 'bunk'],
  required: false,  // some series may not fit any occupancy bucket
}
```

Drives the 4 occupancy hub rosters. Seed values come from the xlsx col F `/bedroom-set/{occupancy}/{series}` entries.

### Categories collection

Add `axis_filter` for facet promotion:

```ts
axis_filter: {
  type: 'json',  // shape: { axis: string, value: string | number }
  required: false,
  admin: {
    description: 'When set, the page auto-filters its product query by this axis value. Use for SEO-promoted facet pages only.',
  },
}
```

Only the 5 facet sub-leaves have this set. Everywhere else, `axis_filter` is null.

### Migration

Single migration: `20260523_xxxxxx_pre_import_schema.ts` — adds both fields. Note the [[zhic-overhaul-handoff|Payload 3 hasMany text persistence quirk]]: `occupancies` will need the DELETE-then-INSERT-via-pg workaround in seed.

### D3 — Series-less rows ("-" and "_other_items") — RESOLVED: defer, set NULL for now

### D4 — Finish-variant encoding — RESOLVED: heuristic approved

Heuristic for the importer:

1. Filename contains `-cream-` or ends `-cream.webp` → axis `finish=cream`
2. Else `-gray-` / `-gray.webp` → `finish=gray`
3. Else `-green-` / `-green.webp` → `finish=green`
4. Else folder name `parla-2staged` → `finish=two-stage`
5. Else default `finish=natural` (or leave NULL for series with no finish split)

Each (series, type) generates one `productVariant` per unique finish detected. The `axes` JSON carries `{ finish: 'cream' }` etc. Spot-check ~10 mappings against actual image files during import-script dev.
