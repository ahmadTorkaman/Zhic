## 4. Locked decisions (carry-forward context)

These were agreed during brainstorming and should NOT be re-litigated in future sub-projects unless the user explicitly revisits.

| Area | Decision |
|---|---|
| **SEO** | Every series → `/designs/[slug]` hub. Every piece-type (category leaf) → `/categories/[slug]` hub. **NO per-variant URLs, ever.** |
| **Piece types** | Use the existing `Categories` collection (not a separate `pieceTypes`). Extended with `allowed_axes`, `rule`, `cover`. Drop the legacy `piece_type` enum on Products. |
| **Variants** | New `productVariants` collection. Flexible `axes: JSON` per variant (NOT hardcoded size/finish/fabric). Validated against parent piece-type's `allowed_axes`. |
| **Series ↔ Designs** | 1:1. Every series has a Design record AND a hub page at `/designs/[slug]`. |
| **PDP slug** | `<series>-<piece-type>` (e.g., `baloot-bed`). |
| **Tree depth** | Full hierarchy — ~7 parent categories AND 32 leaf categories, both with public hub pages. |
| **Categories template** | One template (P1) — middle section swaps between product grid (leaf) and child-category tiles (parent). |
| **Filter UI on categories** | Right-side sticky sidebar on desktop ≥1024px; bottom-sheet drawer on mobile. URLs canonical to base + `noindex,follow` on filtered. |
| **PDP picker placement** | Sticky bottom bar (option C), inline chip strips visible (β density), hero cross-fades to `variant.image` (option 2). |
| **Default variant** | Auto-pick first variant on load (NOT required-selection blocking). |
| **Glass language** | Canonical `.glass-card` recipe (`--glass-bg` 60% ivory + `--glass-blur` 24px + sand 50% border). Sticky bars use 78% ivory for legibility over hero images. **Tiles use "quiet card"** (opaque cream wash, no backdrop-blur) for GPU efficiency on grids. |
| **CTA accent** | Ink background + **2px gold accent line at bottom edge** — gold whispers, not dominates. "Accent is earned" per [[feedback_zhic_luxury_restraint]]. |
| **Animation curves** | `cubic-bezier(0.22, 1, 0.36, 1)` out-soft (primary); `cubic-bezier(0.16, 1, 0.3, 1)` out-quint (slide-ups). |
| **Animation timing** | Hero cross-fade 600ms; price cross-fade 200ms; bar slide-up 350ms. `BlurInText` 700ms per word with stagger 90ms. Site-wide. |
| **Header height** | `--header-height: 56px` exactly. Header is `py-3` desktop + `h-8` icons + `h-7` hamburger. Don't add taller content without bumping the token + every page that pads `var(--header-height)`. |

## 5. Operational gotchas (pitfalls that bit us — avoid in the next session)

1. **`pnpm --filter @zhic/api generate:types` is broken** (tsx + Node 24 ESM bug). Hand-update `services/api/src/payload-types.ts` whenever the Payload config changes. When the upstream fix lands, regenerate cleanly.

2. **Payload 3 hasMany text persistence quirk** — adding/updating an `array` or `hasMany: true` field via `payload.update()` does NOT reliably persist the child rows. Both /categories (`allowed_axes`) and /products (`product-variants_axes`, `categories_allowed_axes`) hit this. Workaround: after the `upsertBySlug`/`update` call, write the child rows directly via Node `pg` (DELETE + INSERT with UUIDs). See `services/api/src/seed.ts` for the established pattern.

3. **`services/api/next.config.ts` has `typescript: { ignoreBuildErrors: true }`** for a pre-existing `sharp` type mismatch. Build passes; typecheck CI step still catches real type errors via `pnpm --filter @zhic/api typecheck` separately. Don't remove the flag without fixing the underlying sharp issue.

4. **Production server caching:** Next.js prod mode caches `.next/cache/fetch-cache/`. After seed changes, you may need to delete that directory + `pm2 restart zhic-web` to see fresh data. The dev server alone won't pick up source changes — code changes require `pnpm --filter @zhic/web build` THEN `pm2 restart zhic-web --update-env`.

5. **Collection slug conventions** — multi-word slugs are kebab-case (`product-variants`, `journal-categories`, `payload-kv`). Single-word slugs match the collection name (`categories`, `designs`, `materials`). The REST endpoint uses the same casing. **DB table names are always snake_case** (`product_variants`, `categories`). Don't mix — the SiteHeader rework session also caught a similar slug/table-name issue.

6. **`payload_locked_documents_rels` column** — when adding a new collection, Payload's Drizzle layer expects a corresponding `<collection>_id` column on this table. If missing, `payload.find()` queries crash. Include in the same migration. See `20260522_150000_create_product_variants.ts` line ~87 for the pattern.

7. **Logical CSS only** — CLAUDE.md mandates `margin-block-start/end`, `padding-inline`, `inset-block-*`, `border-inline-*` etc. Never `margin-top/bottom/left/right`. The `feature-dev:code-reviewer` agent will flag any drift; review and fix immediately.

8. **Persian digits via `@zhic/locale`** — any user-facing number needs `toPersianDigits(...)`. The `formatMoney` helper from `@zhic/money` already does this when called without overrides. Test files don't need it (test data is ASCII).

9. **Mockups are the visual source of truth** — when the spec disagrees with the mockup, the mockup wins. If reading the spec, also open the corresponding mockup URL. They're at `apps/web/public/docs/<name>-mockup.html`.

10. **Test infrastructure for new component tests** — `@testing-library/react` + `jsdom` + `@vitejs/plugin-react` are devDeps. First-time-needed installs are documented in sub-project D Task 5 and sub-project C Task 8.

11. **Header height + page padding** — `--header-height: 56px` is the single source of truth. Pages that need to clear the header use `padding-block-start: var(--header-height)` or `calc(var(--header-height) + var(--space-N))`. **Do not hard-code 72px / 88px** — `/designs` filmstrip ate this twice.
