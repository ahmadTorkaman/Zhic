data model

New field on the home global

home.heroSlides[] — array on the existing home global. Each entry: image (upload→media, required) · alt (text) · link (optional URL or relation).
New collection rooms — three seeded entries for the age categories. Fields: name (e.g. «اتاق کودک»), slug (kid/teen/adult), cover (upload→media), tagline (the line under the title), longDescription (richText for the landing page), plus the standard seoFields.
New route /rooms/[slug] — landing page per age. Initially renders cover + tagline + longDescription; later iterations can surface relevant products / designs (likely via an age field on products, but that's a v2 — flag it in the spec).
New component packages/ui/ParallaxImage — porting framer.com/m/ParallaxImage-prod-3M0qB4.js. Scroll-driven vertical parallax, ~40 lines. Props: src, alt, verticalAmount (default 80), borderRadius. Applied to the three age-category media tiles. Honors prefers-reduced-motion. The mockup above runs the ported logic inline so you can scroll the page and feel the effect.
Home showrooms teaser — compact 3-up card grid between journal and footer. Smaller scale than the journal cards (3:2 cover with top-right curve, condensed type, no excerpt — just city, address, phone). Header row: eyebrow + heading + «فهرست کامل» CTA. Phone version: stacked rows with 96×96 thumbnail + city + address. Auto-populates from the existing showrooms collection, ordered by is_central first then name, limit 3. No schema changes needed; new component packages/ui/HomeShowroomsTeaser.
Home journal teaser — three-row scroll parallax (ported from framer.university). Replaces the 3-up static grid. Three horizontal rows of compact article cards (cover 3:2 + category + 2-line title, no excerpt), each row drifting at a different rate as the user scrolls: Row 1 right @ 0.35×, Row 2 left @ 0.55× (opposite), Row 3 right @ 0.75×. JS listens to vertical scroll, maps section progress to translateX per row via requestAnimationFrame. Honors prefers-reduced-motion. New component packages/ui/HomeJournalRows. Home query shifts from featured:true limit 3 to published limit 15, distributed round-robin into 3 rows of 5 cards. No collection schema changes. Mobile: same 3-row parallax with narrower cards (200px vs 340px) and dampened displacement (max 140px vs 300px) so cards don't run off-screen at small viewport widths.
Footer redesign — bg = --color-forest-dark (shared with about-us). Add a «در تماس باشیم» hero strip above the existing 4 link columns:
· Inline form (name + phone + message) submits via the existing submitInquiry server action — reuse InquiryFormSlim styled for dark bg.
· Contact info block: phone, email, Hamedan address. Add to a siteConfig Payload global (or extend the existing one) with contactPhone, contactEmail, address (richText), hours (text).
· Newsletter signup — new subscribers collection (email, subscribedAt, source) + a POST /api/newsletter route. Mail provider integration deferred (Package 2).
· Socials — array on siteConfig.socials[]: { platform: 'instagram'|'telegram'|'whatsapp'|'aparat', url: string }.
The 4 existing link columns + legal line stay below the strip.
New utility packages/ui/BlurInText — word-by-word blur reveal taken from split-text-mockup.html. JS splits each text element's text nodes into per-word <span class="blur-piece"> at runtime (whitespace preserved as text nodes; existing inline children kept as single pieces). Each word stays a single text run so Persian glyph shaping is intact. CSS: rest state opacity:0; filter:blur(18px); visible state opacity:1; filter:blur(0); transition 700ms ease-out-soft, delay = --bi × 90ms per word index. IntersectionObserver triggers once when each element enters viewport, then adds is-visible to all child word spans. prefers-reduced-motion honored. Elements containing data-count-up are skipped — they animate at the element level instead.
New component packages/ui/CountUp — animated number ticker, ~40 lines. Props: value (number), suffix (optional, e.g. «+»), duration (default 1500ms), className. IntersectionObserver fires once on first scroll-into-view (one-shot, then io.unobserve(el)); requestAnimationFrame animates with ease-out cubic; each frame formats through toPersianDigits from @zhic/locale. prefers-reduced-motion renders the final value immediately. Used in HomeBrandStatement — also requires shifting BrandStat from { value: string } to { value: number; suffix?: string }. Section bg = new --color-forest-dark (#2D3A2E) — dark tone of the system green, shared with the footer (about-us and contact-us match). Add to packages/design-system/css/tokens.css alongside --color-forest.
Auto-rotate cadence, hover-to-pause, swipe on phone, prefers-reduced-motion = implementation notes for the hero carousel.
